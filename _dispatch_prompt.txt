**Project:** C:\Projects\Mater-Browser

## Goal
Make Easy Apply flow bulletproof: if anything goes wrong, close the modal, return FAILED quickly, and let the runner move to the next job. Never hang.

## Changes needed:

### 1. `src/agent/linkedin_flow.py` — Wrap `_process_easy_apply` with overall timeout and per-page error handling

**A) Add a hard timeout wrapper around the entire `_process_easy_apply` method.**

Use a simple wall-clock approach (not threading) — track `time.time()` at start, check elapsed at each loop iteration. If elapsed > EASY_APPLY_TIMEOUT_SECONDS (default 120 seconds), break and return FAILED. Import `time` at top.

Add constant at module level:
```python
EASY_APPLY_TIMEOUT_SECONDS: float = 120.0
```

**B) Wrap each page iteration body in try/except.**

Inside the `for page_num in range(self._max_pages):` loop, wrap the body (from modal hash check through click_next) in a try/except that catches `Exception`. On exception:
- Log the error at WARNING level with the page number
- Increment a `page_errors` counter
- If `page_errors >= 2` consecutive, break the loop
- Otherwise `continue` to retry the page

**C) Add confirmation check immediately after click_next succeeds.**

After `filler.click_next()` returns True, wait 500ms then immediately check `filler.is_confirmation_page()`. If True, close modal and return SUCCESS. This catches the case where Submit was clicked (not just Next), avoiding an unnecessary extra loop iteration.

**D) Always close the modal on exit (success or failure).**

Add a helper method:

```python
def _close_modal(self) -> None:
    """Close Easy Apply modal to clean up for next job."""
    try:
        dismiss_selectors = [
            'button[aria-label="Dismiss"]',
            '[data-test-modal-close-btn]',
            'button.artdeco-modal__dismiss',
        ]
        for selector in dismiss_selectors:
            btn = self._page.raw.locator(selector).first
            if btn.is_visible(timeout=1000):
                btn.click()
                self._page.wait(500)
                return
    except Exception:
        pass
    try:
        self._page.raw.keyboard.press("Escape")
        self._page.wait(500)
    except Exception:
        pass
```

Call this before EVERY return from `_process_easy_apply` that isn't SUCCESS (on success, call after `filler.close_modal()` as fallback).

**E) Handle the "modal not found" case in the loop.**

If `filler.fill_current_modal()` returns False (no modal found), check if it's a confirmation page. If not, break immediately — don't keep looping on a missing modal.

## Code Standards

Write code that looks inevitable. Follow these constraints:

**Restraint**
- Solve it in one file if possible
- No abstractions until the third time you need them
- No classes if functions will do
- No inheritance - use composition

**Functions**
- Max 25 lines, aim for 15
- One level of nesting max
- Name describes exactly what it does: `extract_billable_hours()` not `process_data()`
- Input → transform → output. No side effects unless that's the point.

**Files**
- Max 200 lines for new files
- One clear responsibility
- If you're adding a second "system" to a file, stop and split

**No Ceremony**
- No AbstractFactory, no IServiceProvider, no Manager classes
- No code "just in case" - solve the actual problem
- Delete commented-out code, don't keep it

**Data**
- Use dataclasses or plain dicts, not classes with only __init__ and getters
- Data flows obviously - reader should predict what happens next
- No global state

The best code is code you delete. Every line is a liability.
